# 盛れカルテ Web Demo

盛れカルテ（Selfie Score）のスコアリングロジックをブラウザで動作確認するための PoC。  
実機ビルドが不要で、Next.js 開発サーバーだけで動く。

## 技術スタック

- Next.js 14 (App Router)
- TypeScript + Tailwind CSS
- MediaPipe Face Landmarker (`@mediapipe/tasks-vision`) — 478点ランドマーク

## 起動方法

```bash
cd web-demo
npm install
npm run dev
```

ブラウザで http://localhost:3000 を開く。  
初回アクセス時に MediaPipe モデル（数十 MB）を CDN から読み込む（数秒かかる）。

## 機能

| タブ | 内容 |
|------|------|
| カメラ | フロントカメラでリアルタイム顔検出・スコアリング |
| 画像アップロード | 端末内の写真を選択してスコアリング |

### スコアリング（4軸・100点満点）

| 軸 | 配点 | 算出方法 |
|----|------|---------|
| 構図整合性 | 40 | 顔の画面占有率・左右対称性・ロール角（極端な傾きのみ減点） |
| 光の質 | 25 | 自己申告（暗い〜最高の5段階） |
| 加工適正度 | 20 | 自己申告（なし〜強烈の山型評価） |
| 表情 | 15 | ブレンドシェイプの笑顔度・開眼度から自動計算 |

### 設計方針（企画調査メモより）

- **「絶対角度を正解とする評価は行わない」**: 45〜50度が正解、といった評価はしない。  
  特定の角度に関わらず、技術的失敗（歪み・傾き・画面占有率の異常）のみ減点。
- 加工度は「高いほど良い」ではなく**山型評価**（やりすぎ減点）。
- 黄金比（顔の造形）は評価軸から除外し、撮影テクニックに純化。

## 注意事項

- インターネット接続が必要（MediaPipe モデルを CDN から読み込む）
- カメラ機能は HTTPS または localhost でのみ動作
- iOS Safari でのカメラ動作は実機確認推奨

## ディレクトリ構成

```
web-demo/
├── app/            # Next.js App Router
├── components/
│   └── FaceScorer.tsx   # メインコンポーネント（カメラ・アップロード・スコア表示）
└── lib/
    ├── scoring.ts        # スコアリングロジック（App.tsx から移植）
    └── mediapipeUtils.ts # MediaPipe 検出結果の変換ユーティリティ
```
