# 盛れカルテ（Selfie Score）企画調査メモ

## 1. コンセプト概要

- **愛称**: 盛れカルテ
- **正式名称（仮）**: Selfie Score
- **対象**: ホスト・キャバ嬢・コンカフェ嬢（コスプレイヤー向けは別アプリ案として保留）
- **コンセプト**: 顔そのものの美しさ（黄金比型）ではなく、「自撮りの撮影テクニック」を評価するアプリ。SNS集客に直結する業界向けに特化。
- **プラットフォーム**: Android / iOS（PC不要）。iOSは開発経験が浅いため要注意。

---

## 2. 市場調査

### 日本市場
- 「盛れ度」を直接スコア化する専用アプリは存在しない。
- 既存アプリは大きく2系統:
  1. **顔面偏差値・黄金比系**（FaceScore, FaceChecker）: 顔のパーツが黄金比にどれだけ近いかを0-100点・S〜Eランクで評価。月額500円程度のサブスクモデルが相場。
  2. **写真品質系**（フォト鑑定さん）: 構図・明るさ・色味などをAIが分析し改善アドバイスを出す汎用写真評価アプリ。ポートレート特化ではない。
- → 「自撮りの盛り技術（光・角度・構図）」特化、かつホスト/キャバ向けは空白地帯。

### 海外市場
- **直接競合が存在**: Media.io「AI Selfie Score」は、照明・角度・表情・フレーミング・背景・全体的な印象を分析し、顔の造形ではなく「セルフィー写真そのものの良さ」をスコア化。「顔だけのスコアより明確な答えが得られる」と差別化を明示している。
- PhotoRater, OnPointFresh等も同様の評価軸（lighting, angle, expression, framing, background）を採用。
- ただし英語UIで日本のSNS文化・ホスト文化には未対応。アプリ間の採点ロジックの違いから、絶対値スコアの信頼性には限界があるとの指摘も。

### 黄金比の妥当性について
- 「顔の黄金比1:1.618」は美容業界で広く使われる説明だが、心理学的な再現性には疑問がある（マーケティング的な通説）。
- → 「盛れカルテ」では黄金比（顔の造形評価）を主軸にせず、「撮影条件・加工の最適化度」に絞る方針が妥当。既存アプリとの差別化にもなる。

### 業界ニーズの裏付け
- キャバクラ/ホストクラブのSNS集客戦略では、キャスト個人のアカウントで自分の写真を中心に発信し、ファン獲得・集客につなげるのが基本戦略。
- SNS経由で来店した客は投稿写真の話題で会話が盛り上がるため、「映える1枚」がそのまま営業ツールになる構造。一般消費者よりペインが強い。
- 拡散性の高さゆえ、世界観統一・炎上リスクとの隣接も指摘される。

## 3. 「盛れ」の言語化・現場の知見（ホスト向け記事より）

### 角度・構図
- スマホを斜め上方向に**45〜50度**傾けるのが盛れる角度。
- 「利き顔」（笑うと口角が上がりやすい側）から撮ると顔のバランスが良く見える。
- 真正面・下方向からの撮影は顔に影ができ、輪郭・目鼻立ちがもたついて見える。
- カメラのグリッド線の交点に顔を合わせる三分割法も「自撮り黄金比」として言及。

### 光
- 自然光（太陽光）を活用すると美白肌効果。
- 白色の光・壁・服装はレフ板効果で透明感・清潔感を演出。
- 夜は自販機の光が「盛れる」という現場のインサイトあり。リングライトや白紙ノートでの代用も。

### 加工度
- ホスト業界には「パネルマジック」という用語があり、本人と写真が別人すぎる状態を指す。
- 整形レベルの過度な加工は厳禁。骨格・体型を大きく変えない適切なさじ加減が重要。
- 加工しすぎはSNSで「写真詐欺」として炎上するリスクがある。
- → 加工度は「高いほど良い」ではなく**適正範囲がある山型評価**が必要。

### 撮影シーン・文脈
- 店内だけの撮影は「仕事感」が強くなり、客が抵抗感を覚える可能性。
- 休日の様子や外出先の写真を交えることで、夜のイメージが緩和され一般層からの集客にもつながる。
- → 単一画像のスコアだけでなく、投稿全体のバリエーション・文脈管理という視点も今後の検討候補。

## 4. スコアリング設計案（現時点の暫定版）

### 4.0 評価ロジックの根本方針（重要・確定事項）

- **「絶対角度」を正解とする山型評価は採用しない**。
  - 当初「45-50度の煽り角度が盛れる」という業界知見をアングル評価の基準にする案があったが、これは正面からの宣材写真や「清楚系」「お姉さん系」など、あえて正面・控えめな角度を選ぶ系統と矛盾するため撤回。
  - 45-50度の知見は、スコアに直結しない**アドバイス文の提案**（「もう少し角度をつけると、より小顔に見えるかもしれません」等）として残す。
- **代わりに「構図としての技術的整合性」を評価する**方針。
  - 正面の場合: 左右対称性（目・口角の高さの差分など）、水平が取れているか。
  - 斜めの場合: 三分割法との整合性（顔のパーツが構図のグリッドに収まっているか）。
  - 角度に関わらず: 極端な歪み・不自然な傾き・近すぎ/遠すぎ（画面占有率）といった技術的失敗のみを減点対象とする。
- この方針により、正面写真・斜め写真・系統の異なる写り方を同じ土台で評価できる。

黄金比（顔の造形）を評価軸から外し、撮影テクニック評価に純化。

| 軸 | 配点（暫定） | 算出方法 | 評価の型 |
|---|---|---|---|
| 構図整合性 | 40 | 正面: 左右対称性（口角・目の高さ差）／斜め: 三分割法との距離。共通: 極端な歪み・傾き・画面占有率の異常を減点 | 技術的失敗のみ減点（角度自体は問わない） |
| 光の質 | 25 | 顔領域の輝度ヒストグラムの対称性（陰影の偏り検出） | 偏りが少ないほど高評価 |
| 加工適正度 | 20 | ユーザー自己申告スライダー（0-100、または5段階） | 適正範囲の山型（やりすぎ減点） |
| 表情 | 15 | 口角の角度、目の開き具合（EAR） | 高いほど高評価 |

### 加工度スライダーの位置づけ
- 自動検出ではなく**自己申告スライダー**で持たせる。
- 単独スコアとしてではなく、他の軸への「補正・警告フラグ」として機能させる方針。
  - 加工度が高い領域に入ったら警告バッジ（「実物とのギャップに注意」）。
  - 加工度が低い場合、光・構図のスコアの重要性をより強く反映。
- 履歴として記録し、「いつもの加工度」からの逸脱を可視化するセルフチェック機能としても活用可能。

### 重み付けについて
- 現状のネット調査だけでは、各軸の相対的な重要度（重み付け）の精度を詰めることは不可能。
- 評価軸の妥当性（角度・光・構図・表情・加工度）自体は複数ソースで一致しており確度は高い。
- MVPは仮の重み（均等または角度・光重視）でスタートし、ユーザーの「お気に入り設定」「投稿採用/不採用」の記録機能をアプリに組み込み、データ収集しながら調整する方針。

## 5. 技術調査（モバイル実装）

### 結論: クライアントサイド処理でAPI課金なしの構成が可能

### 推奨スタック

| 用途 | ライブラリ | iOS対応状況 |
|---|---|---|
| カメラ/写真選択 | expo-image-picker, expo-camera | ◎ Expo標準、設定が簡単 |
| 顔検出・ランドマーク・角度（ヘッドポーズ） | react-native-vision-camera + react-native-vision-camera-face-detector | ◎ 実装例あり、ネイティブコード不要 |
| 明るさ/輝度解析 | react-native-skia（Canvas相当） | ◎ |
| Face Mesh（478点・精密） | ML Kit Face Mesh経由 | △ 現時点でiOS未対応 |

### 補足事項
- **expo-face-detectorは廃止済み**（SDK 51以降利用不可）。古い記事の情報は注意。代替はreact-native-vision-camera。
- react-native-vision-camera-face-detectorは、目・鼻・口のランドマーク位置とヘッドポーズ（角度）推定を提供し、コミュニティプラグインによりネイティブコードなしで利用可能。2025年5月時点でiOS実装例あり（react-native-vision-camera, react-native-vision-camera-face-detector, @shopify/react-native-skia, react-native-worklets-core, react-native-reanimated の組み合わせ）。
- 478点の精密なFace Meshは現時点でiOS非対応のプラグインが多いが、「盛れカルテ」の評価軸（角度・三分割法・表情）はML Kitベースの顔検出・ランドマーク・ヘッドポーズで十分カバー可能と判断。
- リアルタイム処理は不要（撮影後の静止画分析が中心のため）。

### 開発方針
- React Native（Expo Dev Client / カスタムビルド）で両OS同時開発。
- iOS未経験への対応として、まずは顔検出してヘッドポーズの数値を画面表示するだけの検証プロジェクトを作成し、スタックの動作確認を行うことを推奨。

### PoC（技術検証）進行状況
- 評価ロジックの確定を待たず並行で着手。「データ取得確認」の範囲（評価ロジック非依存）に絞って実施。
- `morekarte-poc` プロジェクトを作成済み（Expo + react-native-vision-camera + react-native-vision-camera-face-detector構成）。
- 確認内容: landmarks（目・耳・鼻・頬・口の座標）、contours（顔・目・眉・唇の輪郭点）、headEulerAngleX/Y/Z（参考値として記録、スコア主軸にはしない）、開眼度・笑顔度、bounds（画面占有率）、左右の口角・目のY座標差分（左右対称性評価の検証用）。
- セットアップ手順・確認チェックリストは `SETUP.md` に記載。iOS/Android実機ビルドでの動作確認が次のアクション。
- ヒアリング予定: 2026-06-27。

## 6. ヒアリング計画

### 対象
- ホスト、キャバ嬢、コンカフェ嬢（コスプレイヤーは別アプリ案として保留、ヒアリング項目は別途検討）

### 構成（Part 1〜3、15-20分・選択式中心）

**Part 0: 役割の整理**
- 「あなたにとってSNSの写真は、主に何のためのものですか？」（集客/ファン獲得/キャラ表現/自己満足など、複数選択可）

**Part 1: 写真選定の実態**
1. SNSに載せる写真を選ぶとき、何を一番気にするか（自由回答）
2. 1回の撮影で何枚撮るか、採用率はどのくらいか
3. 「盛れた」写真と「微妙」な写真の違いを言語化（実写真の比較が望ましい）
4. 撮影時に意識している角度・ポーズ
5. 「ここなら盛れる」定番スポット・光の条件

**キャバ嬢・コンカフェ嬢向け追加**
- 「系統」（キュート系/クール系/お姉さん系など）を意識した写真選び・系統に合う/合わない写真の違い
- 宣材写真とプライベートSNS投稿での選び方・加工の強さの違い

**Part 2: 加工について**
6. 加工の頻度・使用アプリ/機能
7. 「OK」「やりすぎ」のライン判断基準
8. 加工しすぎて指摘された/反応が悪かった経験
9. シーンによる加工強度の変化（通常投稿 vs 宣材写真）

**Part 3: アプリコンセプトへの反応**
10. スコア軸構成（角度・構図・光・表情・加工度）への意見・不足項目
11. 点数表示 vs アドバイス文表示、どちらが好ましいか
12. 「いつもの加工度」記録・比較機能への反応
13. 仲間内での点数・ランキング比較機能への反応（炎上・競争意識への感覚）
14. 利用シーン（投稿前/宣材撮影時/暇な時など）、使ってみたいか

### 聞き方の工夫
- Part 3で見せるモックには、点数だけでなく「アドバイス文」のサンプルも用意し、トーン感（優しい/厳しい、業界用語の使用感）へのフィードバックも取得。
- 13番はナイーブな質問のため後半に配置。
- 可能であれば本人の投稿写真を見ながら「これは何点だと思う？」と聞き、感覚と数値のズレを把握。

## 7. マネタイズ設計

### 基本方針
- フリーミアム（無料広告モデルは採用しない）。
- 無料モデル先行→後からの有料化は「改悪」と受け取られ離脱リスクが高いため、最初から「無料枠＋課金で解放」という構成にする。
- 無料枠の上限設定が、ターゲット層（ライト層 vs 業界ヘビーユーザー）を自然にセグメントする。

### プラン構成

| 機能 | 無料 | スタンダード | プロ |
|---|---|---|---|
| 診断回数 | 1日3回 | 無制限 | 無制限 |
| 基本5軸スコア | ◯ | ◯ | ◯ |
| 加工度履歴・トレンド表示 | - | ◯ | ◯ |
| 複数枚比較・ランキング | - | - | ◯ |
| 系統別アドバイス（キュート系/クール系等への適合度） | - | - | ◯ |
| 結果のエクスポート（PDF/画像） | - | - | ◯ |
| 「いつもの自分」との差分アラート（パネルマジック警戒） | - | - | ◯ |

### 価格

| プラン | 価格 |
|---|---|
| 無料 | ¥0 |
| スタンダード | 月額380円 |
| プロ | 月額980円 |

### 価格設定の考え方
- スタンダード（380円）: 既存の顔診断系アプリ相場（月額500円）よりはっきり下げ、「お試し有料」のハードルを下げる位置づけ。診断無制限＋加工度トレンドという、個人ユーザーの日常利用向け。
- プロ（980円）: 「個人の娯楽」から「業務ツール」へ明確に壁を作る価格帯。複数枚比較・系統別アドバイス・エクスポート・差分アラートなど、業界ユーザー（ホスト/キャバ/コンカフェ）の実務に直結する機能を集約し、980円への納得感を作る。
- 980円という価格自体は、顔診断アプリ相場（500円）の約2倍だが、サブスク全体の相場（動画/音楽配信が月額1,000円超）を踏まえると高すぎる水準ではない。ただし「無料枠とのギャップ」が体感を決めるため、実際のユーザー行動（無料枠の消費ペース、上位機能への利用意欲）を見て調整する前提とする。

### 無料トライアル
- スタンダード・プロともに7日間無料トライアルを設定。
- プロのトライアルは、業界ユーザーが複数枚比較・系統アドバイスなどの実務価値を体感してから判断できるようにする狙い。
- トライアル終了後の自動課金については、ストアガイドライン上の事前明示が必須（UI設計時に注意）。

### 単発課金（検討中）
- サブスクに抵抗があるユーザー向けに、「追加10回チケット: ¥120」等の都度課金オプションも検討余地あり（撮影会など一時的な集中利用への対応）。

### B2B展開（将来検討）
- 店舗・グループ単位でのプロプラン一括契約（例: 5名分セットで個人契約より割安）。個人課金より単価・継続率が見込める将来の収益源として有望。
